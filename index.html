heat equation — ut = α² uxx  |  separation of variables

JS: Math.sin(Math.PI*x), x*(1-x), x<0.5?x:1-x

solution u(x, t) — all N overlaid
Each line is the partial sum solution u(x,t) using N terms, evaluated at the chosen t. Lighter gray = fewer terms (N=1), black = more terms, dashed black = reference solution (N_ref terms). Lines converging toward the reference shows the series is working. Hover over any line to see its N value and exact u at that x.
L² error vs N
L² error = √∫(u_N − u_ref)² dx, integrated over x ∈ [0,1] at fixed t. Measures the average error across the entire domain. A steep drop means each new term contributes significantly. A flat section means those terms have negligible contribution at this t.
L∞ error vs N (max pointwise)
L∞ error = max over x of |u_N(x) − u_ref(x)|. This is the worst-case error at any single point. L∞ is stricter than L² — a solution can have low L² error but a spike somewhere that L∞ catches. Compare with L² plot: if both drop at the same rate, error is uniformly distributed across x.
absolute pointwise error |u_N(x) − u_ref(x)| over x
Each line shows the error at every x for a given N, on a log scale. The sharp dips to near-zero are points where u_N and u_ref happen to cross (difference = 0, log → −∞). The spike at x=0 and x=1 going to 10⁻²¹ is the boundary condition being satisfied exactly (sin(nπ·0) = sin(nπ·1) = 0). The overall level of each line dropping as N increases confirms convergence.
error table
Both L² and L∞ error shown as absolute values (not percentages). Reduction columns show how much error has been eliminated compared to the N=1 approximation. This is an absolute reduction: error(N=1) − error(N), so units are the same as the error itself.
φ(x) vs u(x, t) — initial condition vs evolved solution
Dotted line: φ(x), the initial temperature distribution at t=0. Solid black: u(x,t) at the chosen t using all N terms. Dashed: reference. The gap between φ(x) and u(x,t) shows how much the heat has diffused by time t. At small t the two lines are close; at large t u(x,t) flattens toward zero.
Aₙ coefficient magnitude
|Aₙ| on a log scale. Each Aₙ = 2∫φ(x)sin(nπx)dx is the weight of the n-th mode in the solution. Fast decay → series converges quickly, few terms needed. Slow decay (shallow slope) → slow convergence, many terms needed. Zero Aₙ for even n means φ(x) is symmetric about x=0.5.
solution convergence — each N as individual plot
Each panel shows u(x,t) using exactly N terms (gray), alongside the reference solution (dashed black). The u_max and u_min values shown in each panel are the range of the N-term solution at this t. As N increases, the gray line should overlap the dashed reference more closely.
absolute pointwise error — each N as separate line
Same as in the error analysis tab but here for reference alongside the 2D solution plots. Hover to see exact error at any x for any N.
full solution surface u(x, t) — N terms
3D surface of u(x,t) over x ∈ [0,1] and t ∈ [0.001, 1], built using all N terms. The front edge (small t) approximates the initial condition φ(x). The surface flattens toward zero as t increases — heat diffusing away. Rotate by dragging. The color encodes u value (Plasma colormap: dark = low, bright = high).
convergence surfaces — one per N
Each panel is the full u(x,t) surface built with exactly N terms. Early panels (small N) may show rippling or incorrect shape near t≈0 where convergence is slowest. Later panels should match the full solution above. The L² error and u_max values shown under each title confirm how close each surface is.
